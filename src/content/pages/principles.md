## Prefer Understanding
*"I would rather understand one protocol deeply than memorize ten frameworks."*

Frameworks cycle rapidly, but core systems mechanics endure. By focusing study on RFC specifications, networking layers, and memory constraints, you build technical fundamentals that never expire.

## Prefer Small Systems
*"Simple systems are easier to secure, maintain, and explain."*

Every library or dependency added represents a potential security exposure and runtime failure point. Minimizing surface area ensures code is readable, predictable, and simple to audit.

## Prefer Writing
*"Writing exposes weak thinking."*

In private drafts, it is easy to assume you understand a design. Forcing yourself to explain an architecture, constraint, or failure in writing exposes gaps in comprehension and sharpens logic.

## Prefer Questions
*"Questions survive longer than answers."*

Technology stacks change, but the core problems of distributed consensus, memory safety, and thread coordination persist. Curating active questions directs focus more effectively than cataloging static knowledge.

## Prefer Long Games
*"Compounding beats intensity."*

Rushing builds leads to technical compromise. Consistent study, clean incremental changes, and long-term codebase maintenance yield compounding quality. Longevity is the ultimate verification.
